核心目标并不是预测某一个事件，而是预测与政府相关的正面、负面信息、信息的发展趋势，以及当前民众关注的热点所在，为政府下一步的决策提供帮助**

不看单条微博有没有关联，不看它们是不是在说同一件事，只看 整体群体情绪、整体负面浓度、整体热度波动。

不需要每条微博互相有关系，只需要**聚合统计**这 5 个宏观指标：

## 1. 每日 / 每小时 讨论量走势 （是否做待定）

每天有多少条微博聊政府、民生

- 突然暴涨 = 隐含社会情绪升温
- 持续走低 = 舆情平稳

## 2. 每日负面情绪浓度（最重要，要做）

负面条数 / 总条数 = 负面占比

## 3. 敏感词热度走势（要做）

频率上涨 = 民间不满情绪在累积

## 4. 高互动内容数量（要做）

高转发、高评论

## 5. 话题聚类（自动把散乱微博分成几大类，要做）

- 医疗民生类
- 教育公平类
- 公职作风类
- 物价生活类
- ...................

## 目标1：舆情分析（基础但必须有）

  > “当前舆情是什么样？”

  ### 具体输出：

  - 情感分布（正 / 负 / 中）或者是一个数值
  - 热门关键词
  - 热门微博（按互动量）

  用到：

  - Apache Spark 做清洗 + 聚合

  ## 目标2：趋势分析

  >  “舆情是怎么变化的？”

  ### 具体输出：

  - 每天情感变化趋势（折线图）
  - 关键词热度变化

  本质：
  时间序列统计（不是预测）

  ## 目标3：趋势预测（核心亮点）

  >  “未来会怎么样？

  预测未来几天“负面舆情比例”**

  ```
  输入：历史情感数据
  输出：未来趋势（上升 / 下降）
  ```

对微博正文的处理

>  第一层（规则过滤）
>     ```
>     长度 + 噪声 + 关键词
>     ```
>    第二层（机器学习）
>     ```
>  TF-IDF + Logistic Regression / TextCNN
>  TF-IDF = 高频（当前文本） + 低频（全局）
>  也就是说：
>  一个词如果在“当前文本很重要”，但在“其他文本很少出现”，它就很关键
>  如何用
>  Tokenizer -> 分词
>  HashingTF -> 词频向量
>  IDF -> 加权
>  最终
>  feature vector -> Logistic Regression
>
>  整体
>  1.Rule Features
>  2.TF-IDF
>  3.VectorAssembler
>  4.ML模型
>
>  特征
>  "text","reposts_count","comments_count","attitudes_count","label"
>  "text_length","spam_keyword_count","promotion_count", "is_viral", "engagement_score",
>   "spam_density", "promotion_density"

>## 3. 按【互动热度等级】分类
>
>```
>reposts_count、comments_count、attitudes_count
>```
>
>- 爆款热度：≥1000
>- 高热度：300～999
>- 中等热度：50～299
>- 低热度：＜50
>
>作用：做**热度舆情分级、热门微博分析**
>
>## 4. 按【文本内容长度】分类
>
>依据字段：`text` 文本长度
>
>- 短文：≤80 字
>- 中等篇幅：81～200 字
>- 长文：＞200 字
>
>作用：看**长短文本分布、垃圾微博是不是都是短文案**
>
>## 5. 按【内容质量 / 是否垃圾】分类
>
>依据字段：你模型的 `prediction`
>
>- 优质正常内容（prediction=1.0）
>- 垃圾营销广告（prediction=0.0）
>
>作用：**过滤统计、垃圾内容占比分析**
>
>## 6. 按【博文特征标签】分类
>
>- 带话题（含 #）
>- 带 @好友
>- 纯文本无特殊标记
>
>作用：分析**发文习惯、营销号是否爱带话题 @人**
